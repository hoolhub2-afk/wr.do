## 1. Vercel 部署

Vercel 是 Next.js 的官方推荐平台，部署简单，自动优化。

### 1.1 准备工作

确保项目根目录有 `vercel.json`：

```json
{
  "buildCommand": "pnpm check-db && pnpm build",
  "installCommand": "pnpm install",
  "functions": {
    "app/api/**/*": {
      "maxDuration": 60
    }
  }
}
```

### 1.2 部署步骤

#### 步骤 1：连接 GitHub 仓库

1. 访问 [Vercel](https://vercel.com)
2. 使用 GitHub 账号登录
3. 点击 **Add New Project**
4. 选择 `oiov/wr.do` 仓库或你 Fork 的仓库

#### 步骤 2：配置项目

| 配置项 | 值 |
|--------|-----|
| Framework Preset | Next.js |
| Build Command | `pnpm check-db && pnpm build` |
| Output Directory | `.next` |
| Install Command | `pnpm install` |

#### 步骤 3：配置环境变量

在 **Environment Variables** 中添加所有必要变量（见[环境变量配置](#5-环境变量配置)）。

#### 步骤 4：部署

点击 **Deploy**，Vercel 会自动构建并部署。

### 1.3 数据库选择

Vercel 本身不提供数据库，需要使用外部托管 PostgreSQL：

| 服务商 | 免费额度 | 说明 |
|--------|----------|------|
| [Neon](https://neon.tech) | 每月 300MB 存储 | 专为 Serverless 设计 |
| [Supabase](https://supabase.com) | 500MB 存储 | 开源 Firebase 替代 |
| [Railway PostgreSQL](#2-railway-部署) | 1GB 存储 | 集成 Railway 平台 |
| [Crunchy Bridge](https://www.crunchybridge.com) | 230MB 存储 | 高可用 PostgreSQL |

### 1.4 常用命令

```bash
# 本地预览 Vercel 构建
vercel

# 部署到预览环境
vercel --preprod

# 部署到生产环境
vercel --prod

# 查看日志
vercel logs
```

### 1.5 常见问题

**Q: 数据库连接超时**
- 检查 `DATABASE_URL` 格式
- 确保数据库允许 Vercel IP 访问（Neon/Supabase 无需配置）

**Q: 构建失败**
- 检查 Node.js 版本是否为 20.x
- 确保环境变量已正确配置
- 查看构建日志排查具体错误

## 2. Railway 部署

Railway 是一个现代化的云平台，支持 Docker 部署，提供简洁的 UI 和 CLI。

### 2.1 注册账号

1. 访问 [Railway](https://railway.app)
2. 使用 GitHub 账号登录
3. 完成邮箱验证

### 2.2 创建项目

#### 方式一：通过 Dashboard

1. 点击 **New Project**
2. 选择 **Deploy from GitHub repo**
3. 选择 `wr.do` 仓库
4. 选择 **Deploy Now**

#### 方式二：使用 railway.json（推荐）

在项目根目录创建 `railway.json`：

```json
{
  "$schema": "https://railway.app/schema.json",
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "pnpm start-docker",
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
```

### 2.3 配置 PostgreSQL 数据库

#### 方式一：Railway 内置 PostgreSQL

1. 在项目 Dashboard 点击 **+ Add Service**
2. 选择 **Database** → **PostgreSQL**
3. Railway 会自动创建数据库并生成 `DATABASE_URL`

#### 方式二：使用 Neon/Supabase

1. 在 Neon/Supabase 创建数据库
2. 在 Railway 的 **Variables** 页面添加 `DATABASE_URL`

### 2.4 配置环境变量

在 Railway Dashboard 的 **Variables** 页面添加以下变量：

```
# 核心变量（必须）
DATABASE_URL=postgresql://...
AUTH_SECRET=your-32-char-secret-key
NEXT_PUBLIC_APP_URL=https://your-app.railway.app
AUTH_URL=https://your-app.railway.app

# 可选登录方式
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
GITHUB_ID=your-github-id
GITHUB_SECRET=your-github-secret

# 邮件服务
RESEND_API_KEY=your-resend-api-key
EMAIL_FROM=your-email@domain.com
EMAIL_FROM_NAME=WRDO
```

### 2.5 部署配置

#### 使用 Dockerfile

Railway 支持自定义 Dockerfile。确保项目根目录有 `Dockerfile`：

```dockerfile
FROM node:20-alpine

WORKDIR /app

RUN npm install -g pnpm

COPY package.json pnpm-lock.yaml ./
RUN pnpm install --frozen-lockfile

COPY . .
RUN pnpm run build

ENV NODE_ENV=production
ENV IS_DOCKER=true

RUN pnpm add npm-run-all dotenv prisma@5.17.0 @prisma/client@5.17.0

EXPOSE 3000

CMD ["pnpm", "start-docker"]
```

### 2.6 常用命令

#### 安装 Railway CLI

```bash
# 使用 npm
npm install -g @railway/cli

# 登录
railway login

# 链接项目
railway link

# 查看日志
railway logs

# 打开项目
railway open

# 执行一次性命令
railway run pnpm db:push

# 重启服务
railway up
```

### 2.7 自定义域名

1. 在 Railway Dashboard 进入 **Settings** → **Domains**
2. 点击 **Generate Domain** 或添加自定义域名
3. 在域名服务商处添加 CNAME 记录指向生成的域名
4. 等待 DNS 生效后，Railway 会自动配置 SSL

### 2.8 资源限制与升级

Railway 免费计划限制：
- 500 小时运行时间/月
- 1GB 磁盘空间
- 1GB 内存

如需更多资源，可在 **Billing** 中升级。

## 3. Render 部署

Render 是一个全栈云平台，提供 Web Services、PostgreSQL、Redis 等服务。

### 3.1 注册账号

1. 访问 [Render](https://render.com)
2. 使用 GitHub 账号登录
3. 完成邮箱验证

### 3.2 创建 PostgreSQL 数据库

1. 点击 **New** → **PostgreSQL**
2. 配置：

| 配置项 | 值 |
|--------|-----|
| Name | wrdo-db |
| Database | wrdo |
| User | wrdo_user |
| Plan | Free |

3. 点击 **Create Database**
4. 记录 **Internal Database URL** 和 **External Database URL**

### 3.3 创建 Web Service

1. 点击 **New** → **Web Service**
2. 连接 GitHub 仓库 `wr.do`
3. 配置：

| 配置项 | 值 |
|--------|-----|
| Name | wrdo |
| Environment | Node |
| Build Command | `pnpm install && pnpm postinstall && pnpm build` |
| Start Command | `pnpm start-docker` |
| Root Directory | `.` |

4. 点击 **Create Web Service**

### 3.4 配置环境变量

在 **Environment** 标签页添加变量：

```
# 核心变量（必须）
DATABASE_URL=postgresql://wrdo_user:password@wrdo-db.internal:5432/wrdo
AUTH_SECRET=your-32-char-secret-key
NEXT_PUBLIC_APP_URL=https://wrdo.onrender.com
AUTH_URL=https://wrdo.onrender.com

# 跳过 Docker 内的数据库检查
SKIP_DB_CHECK=true
SKIP_DB_MIGRATION=true

# 可选登录方式
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
GITHUB_ID=your-github-id
GITHUB_SECRET=your-github-secret

# 邮件服务
RESEND_API_KEY=your-resend-api-key
EMAIL_FROM=your-email@domain.com
EMAIL_FROM_NAME=WRDO
```

### 3.5 配置健康检查

在 **Health Check** 区域设置：
- Health Check Path: `/api/health`
- Timeout: 60 seconds

### 3.6 部署触发

每次推送代码到 GitHub，Render 会自动重新部署。

### 3.7 自定义域名

1. 进入 **Settings** → **Custom Domains**
2. 点击 **Add Custom Domain**
3. 输入你的域名
4. 在域名服务商处添加 CNAME 记录：
   - 主机记录：`www` 或 `@`
   - 记录值：`wrdo.onrender.com`
5. Render 会自动配置 Let's Encrypt SSL

### 3.8 资源计划

| 计划 | 价格 | 内存 | CPU |
|------|------|------|-----|
| Free | $0 | 512MB | 0.5 vCPU |
| Starter | $7 | 1GB | 1 vCPU |
| Pro | $25 | 2GB | 2 vCPU |

### 3.9 常用命令

```bash
# 安装 Render CLI
npm install -g render-cli

# 登录
render login

# 查看服务状态
render service list

# 查看日志
render service logs <service-id>

# 重启服务
render service restart <service-id>
```

### 3.10 注意事项

1. **休眠问题**：免费计划 15 分钟无请求会自动休眠，首次访问需要 30-60 秒唤醒
2. **数据库连接**：使用 `Internal Database URL` 以获得更好性能
3. **构建超时**：确保构建命令在 10 分钟内完成

## 4. Coolify 自部署

Coolify 是一个开源的自托管平台，支持 Docker 部署。

### 4.1 环境要求

| 组件 | 最低要求 | 推荐配置 |
|------|----------|----------|
| CPU | 1 vCPU | 2+ vCPU |
| 内存 | 1GB | 2GB+ |
| 磁盘 | 10GB | 50GB+ |
| Docker | 20.x | 24.x |
| Docker Compose | V2 | V2 |

### 4.2 安装 Coolify

```bash
# 安装 Docker（如果未安装）
curl -fsSL https://get.docker.com | sh

# 安装 Docker Compose
mkdir -p /etc/docker
curl -fsSL https://raw.githubusercontent.com/docker/compose/v2/docker-compose-v2.yml -o /etc/docker/compose.yml

# 安装 Coolify
git clone https://github.com/coollabsio/coolify /opt/coolify
cd /opt/coolify

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，修改必要配置

# 启动 Coolify
docker compose up -d
```

### 4.3 初始配置

1. 访问 `http://your-server-ip:8000`
2. 创建管理员账号
3. 配置 SSH 密钥（用于部署）
4. 配置 Docker Socket

### 4.4 创建项目

1. 点击 **New Project**
2. 输入项目名称 `wrdo`
3. 选择 **Git Repository**
4. 输入仓库地址：`https://github.com/oiov/wr.do`

### 4.5 配置资源

#### 创建 PostgreSQL 数据库

1. 点击 **New Resource**
2. 选择 **Database** → **PostgreSQL**
3. 配置：
   - Name: `wrdo-db`
   - Version: 16
   - Root User: `postgres`
   - Password: 安全的随机密码

#### 创建 Web Service

1. 点击 **New Resource**
2. 选择 **Application** → **Next.js**
3. 配置：

| 配置项 | 值 |
|--------|-----|
| Name | wrdo |
| Build Pack | Nixpacks |
| Build Command | `pnpm install && pnpm postinstall && pnpm build` |
| Start Command | `pnpm start-docker` |
| Port | 3000 |
| Health Check | `/api/health` |

### 4.6 配置环境变量

在 **Environment Variables** 中添加：

```
DATABASE_URL=postgresql://postgres:password@wrdo-db:5432/wrdo
AUTH_SECRET=your-32-char-secret-key
NEXT_PUBLIC_APP_URL=http://your-domain.com
AUTH_URL=http://your-domain.com

SKIP_DB_CHECK=true
SKIP_DB_MIGRATION=true

# OAuth 配置
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
GITHUB_ID=your-github-id
GITHUB_SECRET=your-github-secret

# 邮件服务
RESEND_API_KEY=your-resend-api-key
EMAIL_FROM=your-email@domain.com
```

### 4.7 部署与访问

1. 点击 **Deploy**
2. 等待构建和启动完成
3. 访问 `http://your-server-ip:3000`

### 4.8 配置反向代理（推荐）

使用 Nginx 或 Caddy 作为反向代理：

#### Nginx 配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

#### 配置 SSL（Let's Encrypt）

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 获取 SSL 证书
sudo certbot --nginx -d your-domain.com

# 自动续期测试
sudo certbot renew --dry-run
```

### 4.9 备份配置

1. 进入 **Settings** → **Backups**
2. 启用自动备份
3. 配置备份频率和保留策略
4. 配置远程存储（S3、Backblaze B2 等）

## 5. 环境变量配置

### 5.1 核心变量

| 变量名 | 必填 | 说明 | 示例 |
|--------|------|------|------|
| `DATABASE_URL` | 是 | PostgreSQL 连接字符串 | `postgresql://user:pass@host:5432/db` |
| `AUTH_SECRET` | 是 | NextAuth 密钥（32+字符） | `openssl rand -base64 32` |
| `NEXT_PUBLIC_APP_URL` | 是 | 应用访问地址 | `https://app.domain.com` |
| `AUTH_URL` | 是 | NextAuth 回调地址 | `https://app.domain.com` |

### 5.2 OAuth 登录配置

#### Google OAuth

1. 访问 [Google Cloud Console](https://console.cloud.google.com)
2. 创建项目，启用 Google+ API
3. 创建 OAuth 2.0 凭证
4. 添加授权回调：`https://your-domain.com/api/auth/callback/google`

#### GitHub OAuth

1. 访问 [GitHub Developer Settings](https://github.com/settings/developers)
2. 创建 New OAuth App
3. 设置回调 URL：`https://your-domain.com/api/auth/callback/github`

| 变量名 | 说明 |
|--------|------|
| `GOOGLE_CLIENT_ID` | Google OAuth Client ID |
| `GOOGLE_CLIENT_SECRET` | Google OAuth Client Secret |
| `GITHUB_ID` | GitHub OAuth Client ID |
| `GITHUB_SECRET` | GitHub OAuth Client Secret |
| `LinuxDo_CLIENT_ID` | LinuxDO OAuth Client ID |
| `LinuxDo_CLIENT_SECRET` | LinuxDO OAuth Client Secret |

### 5.3 邮件服务配置

#### Resend

1. 访问 [Resend](https://resend.com)
2. 创建账号并获取 API Key

#### Brevo

1. 访问 [Brevo](https://www.brevo.com)

| 变量名 | 说明 |
|--------|------|

### 5.4 其他配置

## 8. 备份与恢复

### 8.1 数据库备份

#### pg_dump 备份

```bash
# 备份数据库
pg_dump -h host -U user -d database > backup_$(date +%Y%m%d).sql

# 压缩备份
pg_dump -h host -U user -d database | gzip > backup_$(date +%Y%m%d).sql.gz
```

#### 自动备份脚本

创建 `/opt/scripts/backup.sh`：

```bash
#!/bin/bash

BACKUP_DIR="/opt/backups"
DATE=$(date +%Y%m%d_%H%M%S)
DB_CONTAINER="postgres"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
docker exec $DB_CONTAINER pg_dump -U postgres wrdo | gzip > $BACKUP_DIR/wrdo_$DATE.sql.gz

# 删除 7 天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

# 上传到云存储（可选）
# aws s3 cp $BACKUP_DIR/wrdo_$DATE.sql.gz s3://your-bucket/backups/
```

添加定时任务：

```bash
# 编辑 crontab
crontab -e

# 添加每日备份
0 2 * * * /opt/scripts/backup.sh
```

### 8.2 数据库恢复

```bash
# 从 SQL 文件恢复
psql -h host -U user -d database < backup_20240101.sql

# 从压缩文件恢复
gunzip -c backup_20240101.sql.gz | psql -h host -U user -d database
```

### 8.3 Railway 备份

Railway 免费计划不提供自动备份，需要手动导出：

```bash
# 使用 pg_dump 导出
pg_dump "postgresql://user:pass@host:5432/db" > backup.sql
```

### 8.4 Render 备份

Render Pro 计划支持自动每日备份。在 **Settings** → **Backups** 中启用。

### 8.5 Coolify 备份

Coolify 支持自动备份到远程存储（S3、Backblaze B2、Filebase 等）：

1. 进入 **Settings** → **Backups**
2. 选择存储提供商
3. 配置访问凭证
4. 启用自动备份

## 9. 性能优化

### 9.1 构建优化

```dockerfile
# Dockerfile 优化示例
FROM node:20-alpine AS deps
RUN apk add --no-cache libc6-compat
WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN pnpm install --frozen-lockfile

FROM node:20-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN pnpm build

FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production

COPY --from=builder /app/prisma ./prisma
COPY --from=builder /app/.next/standalone ./
```

## 10. 安全建议

### 10.1 环境变量安全

- 使用强随机字符串作为 `AUTH_SECRET`
- 定期轮换 API Keys
- 不要在代码中硬编码敏感信息

### 10.2 数据库安全

- 使用强密码
- 启用 SSL 连接
- 限制数据库访问 IP

### 10.3 HTTPS 配置

- 确保所有环境变量中的 URL 使用 HTTPS
- 启用 HSTS：

```nginx
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
```

```bash
# 保持依赖更新
pnpm update

```

## 11. 故障排查

### 11.1 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 502 Bad Gateway | 服务未启动 | 检查服务状态和日志 |
| 503 Service Unavailable | 数据库连接失败 | 检查 DATABASE_URL |
| NextAuth Error | AUTH_SECRET 未配置 | 配置 AUTH_SECRET |
| Prisma Error | 数据库迁移失败 | 运行 `pnpm db:push` |

### 11.2 调试技巧

1. 查看详细日志
2. 检查环境变量是否正确
3. 验证数据库连接
4. 测试 API 端点

```bash
# 测试数据库连接
pnpm exec prisma db push --accept-data-loss
```
